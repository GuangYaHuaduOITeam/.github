前言

基础入门篇

1.C++ Primer

本书内容是C++大师Stanley B. Lippman丰富的实践经验和C++标准委员会原负责人Josée Lajoie对C++标准深入理解的完美结合，已经帮助全球无数程序员学会了C++。还包含大量来自实战的示例和习题。对C++基本概念和技术全面而且权威的阐述，对现代C++编程风格的强调，使本书成为C++初学者的最佳指南；对于中高级程序员，本书也是不可或缺的参考书。

2.C++ Primer Plus

和上边的区别这本开始说了一些C语言基础，适合零基础，一本经久不衰的C++畅销经典教程；一本支持C++11新标准的程序设计图书。它被誉为“开发人员学习C++的教程，没有之一”！《C++ Primer Plus》可以说是一本面向从未学习过C语言甚到是从未学习过编程的人的入门书籍，关注博主的微 信 公 众 号：C和C加加 回复 “88”即可领取相关电子书和C++教程大全，整个书的结构安排较为合理，难度爬升较慢。如果你是一个从未学过C语言（或者压根没学会C）的读者，那么，我相信这本书更适合你。

3.Accelerated C++

这C++书籍覆盖了和C++ Primer一样的内容，但厚度只有 C++ Primer 的四分之一。

这主要是因为本书面向的不是编程的初学者，而是有其它语言经验的C++初学者。

对于初学者，本书学习曲线稍显陡峭，但对于能克服这一点的学习者而言，它确实非常紧凑的介绍了C++这门语言。

4.《C++编程思想》(Thinking in C++)

介绍了各种让你的C++代码更为优雅,健壮的方法,根据作者自己学习C++的亲身体会编写而成.

5.《C++程序设计原理与实践》(Programming: Principles and Practice Using C++)

C++之父写的C++入门书籍。本书面向没有编程经验的初学者，但相信有编程经验的人也能从本书中学到不少东西。

6.《C++初学者指南》

进阶篇

1.Effective C++

有人说C++程序员可以分为两类，读过Effective C++的和没读过的，在国际上本书所引起的反响，波及整个计算机技术的出版领域，余音至今未绝。几乎在所有C++书籍的推荐名单上，作者高超的技术把握力、独特的视角、诙谐轻松的写作风格、独具匠心的内容组织，都受到极大的推崇和仿效。

2.Effective STL

讲解方式和Effective类似，但内容主要面向于STL，有人说 C++ 是 C With STL，足以可见 STL 的重要性，这本书也强烈推荐看下。

3.《深入浅出设计模式》

4.《设计模式:可复用面向对象软件的基础》

5.《HeadFirst设计模式》

6.《大话设计模式》

进阶加强篇

1.More Effective C++

C++真牛人大手笔，以条款的形式回答了程序员在使用C++时经常提出的“应该注意些什么？”，“应该避免什么？”，“应该怎样去解决？”，“为什么？”等问题，C++程序员想要让自己的水平上更高的档次，此书必看，有一定难度，不适合初学者。

2.Exceptional C++

更多（深入）关于C++的规则，值得一读，有时间的话可以看下，反正也挺薄的。

Markdown语法简介

2022-06-26 周刊

广雅花都信息竞赛周刊第4期-附刊

0.简介

    Markdown是一种可以使用普通文本编辑器编写的标记语言,通过Markdown简单的语法可以使文本内容具有一定的格式,浏览器等工具通过这些格式渲染出具有排版的文本内容.

    当然我们周刊也是用Markdown所编写的,你们所看到的网页上的周刊都是已经渲染好的而不是原始文本,这是Markdown区别于常见的Word文档等的区别.也正是因为Markdown独特的编写-渲染分开的方式,使Markdown在网页中非常常见,并且一个Markdown文件在支持Markdown格式的地方,都可以获得很好的渲染.我们可以使用Markdown来编写博客,邮件,还可以很方便的将Markdown文件转化成HTML或者PDF,甚至Github的评论,项目中的readme都是由Markdown编写的.

    Markdown文件的扩展名为.md,内容是常规的文本.

1.语法

1.1 标题

    Markdown一共有6级标题,一级标题需要在相应的文字前加”#“,二级标题加”##“,依此类推,注意”#“与后面的标题内容需要有一个空铬,多余的空格会显示为标题内容:

正确示范:

二级标题

六级标题

错误示范:

#一级标题

##二级标题

###三级标题

####四级标题

#####五级标题

######六级标题

注意上面两种写法的渲染差异.

2022-06-12 周刊

0.前言

    在C++中,数组可以说是我们最常使用的一个数据结构了,从数组出发,我们可以方便的实现各种复杂数据结构.但是C++的传统数组也有其不便之处,如数组大小不可改变,操作下标容易发生设计失误,以及数组操作不便等等问题.

    在传统数组使用不便的情况下,C++中两个最重要的STL容器也随之诞生了:vector(向量)和list(链表).这两个容器不仅大小可变,还继承了传统数组的灵活性,同时提供了大量内置操作供我们使用.可以说,在绝大多数题目中,这两个容器都能很好的替代传统数组.

    这篇周刊的知识量很大,建议准备好笔记本:)

1.基础知识-迭代器简介

1.0 迭代器简介

    在学习vector和list之前,我们首先要来了解STL容器中的重要内容:迭代器.

    简单来说,迭代器是一种专门为STL容器设计的指针,但迭代器与一般的指针相比,具有安全性更强,更直观,而且在使用STL容器时,迭代器也是必不可少的内容.

    也就是说,我们可以将迭代器看作一个复杂了一些的指针.可以用来检查容器内元素并遍历元素.

    C++迭代器分为多种,不同的STL容器支持的迭代器种类会随容器特性的不同而不同,在此处仅介绍竞赛中常用的两种:随机访问迭代器和双向迭代器,下表是STL容器对这两种迭代器的支持情况:

list 双向迭代器

map 双向迭代器
multiset 双向迭代器

stack 不支持

deque 随机访问迭代器

如何提高C++中cout的输出效率

2022-06-03 周刊

广雅花都信息竞赛周刊第1期-附刊

0.前言

    在C++中添加了cout这种输出方式，在易用性和安全性上大大超越了C语言对应的printf函数。但是这也带来了一定的代价：cout的输出效率远低于printf。在一些输出相当大量的题中，这一速度问题有可能导致TLE点的出现；在这种情况下我们一般只能选择使用更快的printf进行输出，这就又导致了一个问题的出现：使用printf进行输出有可能会导致各种疑难杂症。那么如何在这种情况下使用cout来进行高效率的输出？

1.使用”\n”或’\n’而非endl进行换行

    使用方式:

cout为了保证每输出一行就显示的同步性,在endl中加入了换行并刷新输出缓存的逻辑,这一方式保证了cout输出的语句立刻就能出现在输出中,但是频繁的刷新输出缓存也会带来效率上的降低.因此使用"\n"来代替endl可以提升输出速度.

2.ios::sync_with_stdio()函数

    使用方式:

在C++中，由于printf和cout在输出是都要操作输出缓冲区，故为了避免同时使用两者输出是出现的同时读写缓存内容导致输出顺序等的错乱，cout在实现时特意与printf保持同步以规避此问题；但是这一解决方案导致了cout的效率的大幅下滑，故使用ios::sync_with_stdio(false)函数关闭这一同步后，cout的输出效率得以大幅提升。(其实这一招也能提升cin的速度)

3.cin.tie()和cout.tie()函数

    使用方式:

在C++中,为了确保cin和cout的输入输出互不干扰,cin和cout会互相绑定在一起,用tie(0)解绑可以提升输入输出速度.

STL容器系列之queue和stack容器的使用

2022-06-03 周刊

广雅花都信息竞赛周刊第1期

0.简介

    我们在写代码时,常常要使用一些数据类型,其中队列和栈是尤其常用的两种.在C++中,我们一般会用数组+指针的形式来实现这两者,这种方式的好处是十分灵活,但坏处就是灵活性太高带来的各种错误.这种情况下我们就可以使用STL包装的两个容器queue(队列)和stack(栈)来代替数组+指针的形式,来提高编程的安全性.

1.queue部分

1.0 使用准备和初始化

    头文件嘛…bits/stdc++.h天下第一!

    这里先声明一点,就是我们的万能头文件包含所有常用的STL容器相关头文件,使用万能头文件完全够用了.

    当然如果你不放心,也可以手动包含相应头文件,头文件名一般与相关STL容器类型名相同(即queue对应的头文件是<queue>,vector对应的是<vector>,依此类推)

    队列初始化的格式大致可概括为queue<存储类型> 变量名,其中的变量名与我们平时定义变量的变量名的命名规则是一致的(我们一般命名为q开头的队列),而存储类型是指队列中存储变量的类型,既可以是数字(如int, double),字符串(string),布尔(bool),还有自己定义的结构体,类,甚至可以是STL容器(如stack),灵活定义存储类型是使用队列的一大重点.
