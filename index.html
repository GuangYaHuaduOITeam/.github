C++学习书籍推荐

2022-07-03 周刊

广雅花都信息竞赛周刊第5期-附刊

前言

基础入门篇

1.C++ Primer

本书内容是C++大师Stanley B. Lippman丰富的实践经验和C++标准委员会原负责人Josée Lajoie对C++标准深入理解的完美结合，已经帮助全球无数程序员学会了C++。还包含大量来自实战的示例和习题。对C++基本概念和技术全面而且权威的阐述，对现代C++编程风格的强调，使本书成为C++初学者的最佳指南；对于中高级程序员，本书也是不可或缺的参考书。

2.C++ Primer Plus

和上边的区别这本开始说了一些C语言基础，适合零基础，一本经久不衰的C++畅销经典教程；一本支持C++11新标准的程序设计图书。它被誉为“开发人员学习C++的教程，没有之一”！《C++ Primer Plus》可以说是一本面向从未学习过C语言甚到是从未学习过编程的人的入门书籍，关注博主的微 信 公 众 号：C和C加加 回复 “88”即可领取相关电子书和C++教程大全，整个书的结构安排较为合理，难度爬升较慢。如果你是一个从未学过C语言（或者压根没学会C）的读者，那么，我相信这本书更适合你。

3.Accelerated C++

这C++书籍覆盖了和C++ Primer一样的内容，但厚度只有 C++ Primer 的四分之一。

这主要是因为本书面向的不是编程的初学者，而是有其它语言经验的C++初学者。

对于初学者，本书学习曲线稍显陡峭，但对于能克服这一点的学习者而言，它确实非常紧凑的介绍了C++这门语言。

4.《C++编程思想》(Thinking in C++)

介绍了各种让你的C++代码更为优雅,健壮的方法,根据作者自己学习C++的亲身体会编写而成.

5.《C++程序设计原理与实践》(Programming: Principles and Practice Using C++)

C++之父写的C++入门书籍。本书面向没有编程经验的初学者，但相信有编程经验的人也能从本书中学到不少东西。

6.《C++初学者指南》

进阶篇

1.Effective C++

有人说C++程序员可以分为两类，读过Effective C++的和没读过的，在国际上本书所引起的反响，波及整个计算机技术的出版领域，余音至今未绝。几乎在所有C++书籍的推荐名单上，作者高超的技术把握力、独特的视角、诙谐轻松的写作风格、独具匠心的内容组织，都受到极大的推崇和仿效。

2.Effective STL

讲解方式和Effective类似，但内容主要面向于STL，有人说 C++ 是 C With STL，足以可见 STL 的重要性，这本书也强烈推荐看下。

3.《深入浅出设计模式》

4.《设计模式:可复用面向对象软件的基础》

5.《HeadFirst设计模式》

6.《大话设计模式》

进阶加强篇

1.More Effective C++

C++真牛人大手笔，以条款的形式回答了程序员在使用C++时经常提出的“应该注意些什么？”，“应该避免什么？”，“应该怎样去解决？”，“为什么？”等问题，C++程序员想要让自己的水平上更高的档次，此书必看，有一定难度，不适合初学者。

2.Exceptional C++

更多（深入）关于C++的规则，值得一读，有时间的话可以看下，反正也挺薄的。

3.More Exceptional C++

讲到了Exceptional C++没有涉及到的更高级的异常安全技术, 同时讨论了高效的 C++ OOP 方式和如何正确的使用 STL。

4.Exceptional C++ Style

讨论了泛型编程、最优化和资源管理，本书出彩之处在于谈到了如何用非成员函数和单职责原则编写模块化的 C++ 代码。

5.《C++编程规范》(C++ Coding Standards)

“编程规范”这里并不是”代码缩进要用几个空格”，这本书包含了101个例子、惯用法、缺陷，通过这些可以帮助你编写正确、清晰高效的C++代码。

6.《C++模板完全指南》(C++ Templates)

这本书是关于C++11之前的模板的，它覆盖了从非常基础到最高级的元编程知识，解释了模板工作原理的细节(概念和实现方式），并且讨论了大量的缺陷，想学习模板元编程的同学不要错过啰。

7.Beyond the C++ Standard Librart(Boost)

8.《C和C++安全编码》

9.Effective Modern C++

10.《STL源码剖析》(The Annotated STL Sources)

高级篇

1.《C++设计新思维-泛型编程与设计模式之应用》(Modern C++ Design)

泛型编程鼻祖级书籍。本书先介绍了基于策略（policy-based)的设计、type lists 和泛型编程基础。关注博主的微 信 公 众 号：C和C加加 回复 “88”即可领取相关电子书和C++教程大全，然后讲到了许多有用的设计模式(包括small object allocators, functors, factories, visitors, and multimethods) 如何被高效、模块化、清晰的泛型代码实现！

2.《C++模板元编程》(C++ Template Metaprogramming)

更多的是讲解boost::mpl，想要深入理解mpl的可以看一下

3.C++ Concurrency in Action

这本书主要内容是C++11的并发支持，包括线程库、原子(atomics)库、内存模型、锁和互斥量。同时也讲解了开发和调试多线程程序的一些难题。

4.Advanced C++ Programming Styles and Idioms

它确实很老了，可能有些不符合现代的 C++，讲解了一些C++特有的惯用法.，它确实是一本不错的书籍，如果时间闲暇也可以读。

5. Real-Time C++ Effcient Object-Oriented and Template Microcontroller Programming

6. Advanced Metaprogramming in Classic C++

7.《x86汇编语言:从实模式到保护模式》

8.《C++模板元编程实战:一个深度学习框架的初步实现》

参考手册篇

1.《C++程序设计语言》(The C++ Programming Language)

C++之父写的经典 C++书籍，内容覆盖C++的所有东西，从语言内核到标准库、编程范式和语言哲学，2013年5月出版的第四版涵盖了 C++11 的内容。

2.《C++标准程序库》(C++ Standard Library Tutorial and Reference)

这是C++标准库（STL）的引导和手册。 2012年4月发行的第二版涵盖了C++11。

3.The C++ IO Streams and Locales

除了这本书，市面上基本没有讲解 streams and locales 的书。

4.The C++ Standard (INCITS/ISO/IEC 14882-2011)

作者：C++标准委员会，这当然是 C++ 最权威的标准，C++ 标准文档一般是那些“语言律师”比较爱看的，一般使用者应该很少看这种语法大全。

5.Overview of the New C++ (C++11/14)

Scott Meyers是C++社区最受尊敬的作者之一，虽然内容比较简短，但质量却极高。

内容大致覆盖了C++ 11/14标准的内容.

6.The Standard C Library

内容详尽地描述了C语言的标准库.

algorithm头文件

2022-06-26 周刊

广雅花都信息竞赛周刊第4期

0.前言及介绍

    我们在很多大佬的代码中或者洛谷上的各种整活题解中都能看到很多不知所云的各种函数.我们在惊叹这些各种函数是如何是我们的编程工作更加简洁和优雅的同时,也苦于自己只会一个sort() (当然有很多人sort()都用不明白),为了解决帮助上述群众,这篇文章就来讲讲这些函数背后的STL库-<algorithm>头文件.

    algorithm头文件中包含了很多有用的函数,这些函数大致可以分为以下几类(部分译名可能不准确):

  • 非修改性队列操作(Non-modifying sequence operations)

  • 修改性队列操作(Modifying sequence operations)

  • 划分操作(Partitioning operations)

  • 排序操作(Sorting operations)

  • 在已排序队列上的二分搜索操作(Binary search operations (on sorted ranges))

  • 其他已排序队列上的操作(Other operations on sorted ranges)

  • 在已排序队列上的集合操作(Set operations (on sorted ranges))

  • 堆操作(Heap operations)

  • 最值操作(Minimum/maximum operations)

  • 比较操作(Comparison operations)

  • 排列操作(Permutation operations)

  • 数值运算(Numeric operations) (定义位于头文件numeric内)

  • 未初始化内存上的操作(Operations on uninitialized memory) (定义位于头文件memory内)

  • C库算法(C library algorithms)

    不要被这些专有名词和长篇大论吓到了,今天我们只会介绍这些函数中竞赛代码最常用的几个.

    注意:algorithm中的函数虽然基本兼容传统数组,但是algorithm与STL容器搭配起来才是最香的!在阅读本文之前,你最好确保你阅读了前几期讲解STL容器的周刊,并理解了其中的STL容器和迭代器的概念.

    本文下方的完整函数表基本按照C++ Reference的分类和列表撰写,但是没有列出C++20中对各个函数的ranges版本,如果有兴趣了解的可以查询相关资料.(也就是说我们的函数表只截止到C++17标准)

0.简介

    Markdown是一种可以使用普通文本编辑器编写的标记语言,通过Markdown简单的语法可以使文本内容具有一定的格式,浏览器等工具通过这些格式渲染出具有排版的文本内容.

    当然我们周刊也是用Markdown所编写的,你们所看到的网页上的周刊都是已经渲染好的而不是原始文本,这是Markdown区别于常见的Word文档等的区别.也正是因为Markdown独特的编写-渲染分开的方式,使Markdown在网页中非常常见,并且一个Markdown文件在支持Markdown格式的地方,都可以获得很好的渲染.我们可以使用Markdown来编写博客,邮件,还可以很方便的将Markdown文件转化成HTML或者PDF,甚至Github的评论,项目中的readme都是由Markdown编写的.

    Markdown文件的扩展名为.md,内容是常规的文本.

1.语法

1.1 标题

    Markdown一共有6级标题,一级标题需要在相应的文字前加”#“,二级标题加”##“,依此类推,注意”#“与后面的标题内容需要有一个空铬,多余的空格会显示为标题内容:

正确示范:

一级标题

二级标题

三级标题

四级标题

五级标题
六级标题

错误示范:

#一级标题

##二级标题

###三级标题

####四级标题

#####五级标题

######六级标题

注意上面两种写法的渲染差异.

1.2 字体

1.2.1 加粗

    通过在一段文字前后各加两个“*“来对该段文字进行加粗:

Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua. Ut enim ad minim veniam, quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo consequat. Duis aute irure dolor in reprehenderit in voluptate velit esse cillum dolore eu fugiat nulla pariatur. Excepteur sint occaecat cupidatat non proident, sunt in culpa qui officia deserunt mollit anim id est laborum.

1.2.2 斜体

    通过在一段文字前后各加一个“*“来将该段文字变为斜体:

Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.     通过在一段文字前后各加两个”~“来加上删除线:

0.前言

    在C++中,数组可以说是我们最常使用的一个数据结构了,从数组出发,我们可以方便的实现各种复杂数据结构.但是C++的传统数组也有其不便之处,如数组大小不可改变,操作下标容易发生设计失误,以及数组操作不便等等问题.

    在传统数组使用不便的情况下,C++中两个最重要的STL容器也随之诞生了:vector(向量)和list(链表).这两个容器不仅大小可变,还继承了传统数组的灵活性,同时提供了大量内置操作供我们使用.可以说,在绝大多数题目中,这两个容器都能很好的替代传统数组.

    这篇周刊的知识量很大,建议准备好笔记本:)

1.基础知识-迭代器简介

1.0 迭代器简介

    在学习vector和list之前,我们首先要来了解STL容器中的重要内容:迭代器.

    简单来说,迭代器是一种专门为STL容器设计的指针,但迭代器与一般的指针相比,具有安全性更强,更直观,而且在使用STL容器时,迭代器也是必不可少的内容.

    也就是说,我们可以将迭代器看作一个复杂了一些的指针.可以用来检查容器内元素并遍历元素.

    C++迭代器分为多种,不同的STL容器支持的迭代器种类会随容器特性的不同而不同,在此处仅介绍竞赛中常用的两种:随机访问迭代器和双向迭代器,下表是STL容器对这两种迭代器的支持情况:

容器 支持的迭代器类型
vector 随机访问迭代器
list 双向迭代器
set 双向迭代器
map 双向迭代器
multiset 双向迭代器
multimap 双向迭代器
stack 不支持
queue 不支持
priority_queue 不支持
deque 随机访问迭代器

(注意上表中不支持的几个容器)

2.vector部分

2.0 简介

    vector容器的特点基本与C++传统数组一致,其元素存储在内存中也是连续的,所以访问vector中任意元素的用时也是相同的,但是插入和删除元素花费的时间都是O(n)级的.所以在使用vector的场合不宜出现大量插入/删除操作,否则会对性能产生较大影响.
