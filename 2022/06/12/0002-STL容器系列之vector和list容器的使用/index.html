    0.前言

        在C++中,数组可以说是我们最常使用的一个数据结构了,从数组出发,我们可以方便的实现各种复杂数据结构.但是C++的传统数组也有其不便之处,如数组大小不可改变,操作下标容易发生设计失误,以及数组操作不便等等问题.

        在传统数组使用不便的情况下,C++中两个最重要的STL容器也随之诞生了:vector(向量)和list(链表).这两个容器不仅大小可变,还继承了传统数组的灵活性,同时提供了大量内置操作供我们使用.可以说,在绝大多数题目中,这两个容器都能很好的替代传统数组.

        这篇周刊的知识量很大,建议准备好笔记本:)

    1.基础知识-迭代器简介

    1.0 迭代器简介

        在学习vector和list之前,我们首先要来了解STL容器中的重要内容:迭代器.

        简单来说,迭代器是一种专门为STL容器设计的指针,但迭代器与一般的指针相比,具有安全性更强,更直观,而且在使用STL容器时,迭代器也是必不可少的内容.

        也就是说,我们可以将迭代器看作一个复杂了一些的指针.可以用来检查容器内元素并遍历元素.

        C++迭代器分为多种,不同的STL容器支持的迭代器种类会随容器特性的不同而不同,在此处仅介绍竞赛中常用的两种:随机访问迭代器和双向迭代器,下表是STL容器对这两种迭代器的支持情况:

    容器 支持的迭代器类型
    vector 随机访问迭代器
    list 双向迭代器
    set 双向迭代器
    map 双向迭代器
    multiset 双向迭代器
    multimap 双向迭代器
    stack 不支持
    queue 不支持
    priority_queue 不支持
    deque 随机访问迭代器

    (注意上表中不支持的几个容器)

    注意一点,容器名和数组名最大的区别就是容器名是不能当作指针来使用的,所以访问容器的始末时需要使用函数.begin()或.end()来获取指向始末的迭代器,再通过用*迭代器的方式访问迭代器所指向的元素:

    2.vector部分

    2.0 简介

        vector容器的特点基本与C++传统数组一致,其元素存储在内存中也是连续的,所以访问vector中任意元素的用时也是相同的,但是插入和删除元素花费的时间都是O(n)级的.所以在使用vector的场合不宜出现大量插入/删除操作,否则会对性能产生较大影响.

    2.2 常用函数

    函数 参数 用途
    assign() 对vector进行赋值
    back() 返回最后一个元素的值
    front() 返回第一个元素的值
    [i] 即下标,访问第i个元素

    empty() 判断vector是否为空
    pop_back() 删除最后一个元素

    insert() 插入一个元素
    size() 无 返回vector元素个数
    capacity() 无 返回vector可以容纳的元素个数

    swap() 另一个vector 将两个vector的元素整体交换
